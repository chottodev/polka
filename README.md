# Polka

HTTP-сервис с **placeholder-контентом**: SVG-картинки заданного размера, аватары с инициалами, фоновые паттерны. Ответы отдаются как `image/svg+xml`, без нативных библиотек для растра — всё рисуется чистым SVG.

Цель проекта — вырастить до «толстого» placeholder API: аватары разных типов (люди, палитры, звери, инкогнито), простые WxH-блоки, прокси/ресайз стоковых фото и отдельный режим **картинка с текстом** (типографика/растр по мере необходимости).

## Быстрый старт (локально)

Монорепозиторий (**npm workspaces**): `npm install`, затем `npm run build` (сборка UI в `apps/server/public`), `npm start` (только сервер, точка входа `apps/server/index.js`). Разработка UI: `npm run dev -w @polka/ui` (прокси на сервер для `/wxh`, `/docs`, …) и в другом терминале `npm start`.

Порт по умолчанию: `4700` (см. `packages/config`, `.env.example`).

Образ Docker: `TAG=latest ./build.sh`, затем `docker run -p 4700:4700 polka:latest`.

Подробный план: [docs/PLAN.md](docs/PLAN.md).

## API

| Метод | Путь | Назначение |
|--------|------|------------|
| GET | `/health` | Проверка живости |
| GET | `/openapi.json` | Сводная OpenAPI 3 |
| GET | `/docs` | Swagger UI |
| GET | `/wxh/:width/:height` | SVG плейсхолдер; query: `bg`, `fg`, `text`, `rounded`, `font` (размер, px), `family` (гарнитура, 15 пресетов), `gradient` |
| GET | `/wxh/:width/:height/:text` | То же, подпись в path (URL-encoding); те же query |
| GET | `/avatars/initials/:text` | SVG-аватар по инициалам; query: `size`, `palette`, `family`, `font`, `bg`, `fg`, `seed` |
| GET | `/avatars/vector/:kind` | SVG-векторный аватар; `kind`: `man|male|female|cat|dog|panda|fox`; query: `size`, `palette`, `style`, `bg`, `fg`, `seed`, `blush` (`auto|on|off`, для `man`) |

Паттерны (`generatePattern`) в генераторе (`packages/generator/generator.js`) пока **не** выведены на отдельный HTTP-префикс.

## Фичи: что задумано и что есть сейчас

Легенда: **готово** · **частично** · **не сделано**

### 1. Аватарки

| Фича | Статус |
|------|--------|
| SVG, без внешних зависимостей для растра | **готово** (генератор) |
| Инициалы (до 3 символов), цвета `bg` / `fg`, размер | **готово** |
| Мужские / женские / нейтральные векторные аватары | **готово** |
| Палитры / гаммы (наборы цветов по теме) | **готово** (`soft/vivid/earth/mono/ocean`) |
| Зверушки (иллюстрированные аватары) | **готово** (`cat/dog/panda/fox`) |
| Режим инкогнито (нейтральная заглушка) | **не сделано** |
| Конструктор URL (страница или док с превью параметров) | **готово** (Vue: WxH + Аватары) |

### 2. Placehold-картинки (просто квадрат/прямоугольник WxH)

| Фича | Статус |
|------|--------|
| Произвольные width × height | **готово** (префикс `/wxh/`) |
| Подпись, фон, скругление, линейный градиент | **готово** |
| Выдача PNG/JPEG вместо SVG | **не сделано** |

### 3. Стоковые картинки с ресайзом

| Фича | Статус |
|------|--------|
| Источник стока, лицензии, кэш | **не сделано** |
| Изменение размера по WxH / fit / crop | **не сделано** |

### 4. Генератор «картинка с текстом»

| Фича | Статус |
|------|--------|
| Центрированный текст на SVG-плейсхолдере | **частично** (есть в `generatePlaceholder`) |
| Отдельный endpoint / режим под длинные строки, переносы, выбор шрифта | **не сделано** |
| Растровый рендер (canvas/sharp и т.п.) при необходимости | **не сделано** |

### Инфраструктура и качество

| Фича | Статус |
|------|--------|
| Express-маршруты и CORS | **частично** (код есть, зависимости и путь к генератору нужно довести) |
| Dockerfile | есть в репо, не описан здесь детально |
| Тесты, CI | **не сделано** |

## План реализации по секциям (черновик для совместной доработки)

Ниже — опорная структура; порядок и детали можно согласовать отдельно.

1. **Секция «Аватары»** — расширить модель параметров (тип: `initials` \| `human-m` \| `human-f` \| `animal` \| `incognito` \| …), наборы палитр, SVG-шаблоны или спрайты для зверей, единый префикс API и кэш заголовков.
2. **Секция «WxH placehold»** — стабилизировать текущий SVG API, лимиты размеров, документация примеров; опционально второй формат ответа (растр).
3. **Секция «Сток + resize»** — выбор провайдера, проксирование, политика кэша, единый URL вида `/stock/.../WxH`, обработка ошибок и таймаутов.
4. **Секция «Текст на картинке»** — отдельный генератор (SVG достаточно или растр): шрифты, перенос строк, отступы, контраст; возможно отдельный endpoint.
5. **Секция «DX»** — конструктор URL, README/OpenAPI, монорепо: `apps/server` ↔ `packages/generator`, npm-скрипты, docker-compose для локали.

Если нужно, следующим шагом можно пройтись по каждой секции и зафиксировать приоритеты (MVP → v1 → nice-to-have) и форматы URL.
