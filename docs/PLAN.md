# Polka — план работ

Документ фиксирует порядок работ: сначала организационные задачи, затем **секция 1 — WxH placehold** (конфигуратор в UI + API). Стек: **npm workspaces**, **Vue 3 (Composition API)** для UI, один **Express**-процесс, который отдаёт API и статическую сборку UI, поставка **Docker image**, описание API — **OpenAPI** с интеграцией через **`express-openapi`** ([npm: express-openapi](https://www.npmjs.com/package/express-openapi)).

---

## Принятые архитектурные решения

### Маршруты и express-openapi

- Хендлеры лежат в каталоге **`apps/server/routes`** (или эквивалент), подключаемые через опцию в духе `paths: path.join(__dirname, 'routes')` у `express-openapi` — как в документации пакета для автосвязывания операций из спеки с файлами роутов.
- Спецификация OpenAPI остаётся каноническим описанием путей и параметров; версии и нюансы API сверяем с актуальной major-версией `express-openapi`.

### Префиксы по фичам и Swagger UI

- URL **не** монолитный `/api/v1/...`: у каждой фичи свой префикс. Для WxH placeholder: **`/wxh/`** (например `GET /wxh/:width/:height`, при необходимости `GET /wxh/:width/:height/:text`).
- Другие фичи позже получат свои префиксы (аватарки, сток и т.д.).
- **Swagger UI** подключаем (например **`/docs`**) и настраиваем на ту же OpenAPI-спеку.

### SPA и fallback (п. 3) — согласовано

- **Решение:** Vue **SPA**, роутер в режиме **history**; на Express — **fallback на `index.html`** для клиентских путей, не совпадающих с API, `/docs` и реальными статическими файлами. **SSR не делаем.**

Фронт — одностраничное приложение: один `index.html`, переключение экранов — **Vue Router** (URL вида `/wxh`, `/about` без `#`). Без fallback запрос `GET /wxh` дал бы 404, так как на диске нет файла с таким путём; отдаём `index.html`, дальше роутер Vue показывает нужную страницу.

### Конфигурация: `packages/config` + dotenv

- Общие переменные окружения читаются через пакет **`packages/config`**: загрузка **dotenv** из **`.env` в корне репозитория** (путь к корню вычисляется из расположения пакета), плюс при необходимости стандартный `dotenv.config()` для локальных переопределений из текущего каталога.
- Пример ключей: `PORT`, `PUBLIC_ORIGIN`, **`PLACEHOLDER_MAX_WIDTH`**, **`PLACEHOLDER_MAX_HEIGHT`**. См. **`.env.example`** в корне.
- Сервер и (при необходимости) генератор импортируют `const { config } = require('@polka/config')` или workspace-алиас после объявления зависимости в `apps/server`.

### Лимиты и безопасность

- Жёсткий потолок **W** и **H** задаётся конфигом (`PLACEHOLDER_MAX_WIDTH` / `PLACEHOLDER_MAX_HEIGHT`); валидация в API до вызова генератора.
- **Rate limiting** на первом этапе **не** вводим.

### Сборка Docker и тег

- Скрипт **`build.sh`** в корне репозитория: собирает образ, тег задаётся переменной окружения **`TAG`** (по умолчанию `latest`). Опционально **`IMAGE`** (по умолчанию `polka`).

  Примеры:

  ```bash
  TAG=1.2.3 ./build.sh
  IMAGE=myregistry/polka TAG=1.2.3 ./build.sh
  npm run docker:build   # TAG=latest, IMAGE=polka
  ```

## Фаза A — Организация репозитория и инфраструктуры

### A1. Monorepo (npm workspaces)

- Корневой `package.json`: `private`, `workspaces`: `apps/*`, `packages/*`.
- **`packages/generator`** (`@polka/generator`) — чистая логика SVG (лимиты W/H принимать аргументами или импортировать из `@polka/config`, без циклических зависимостей: предпочтительно передавать `maxWidth`/`maxHeight` из сервера).
- **`packages/config`** (`@polka/config`) — dotenv + экспорт `config` (уже заведён черновик).
- **`packages/ui`** (`@polka/ui`) — Vue 3 + Vite, сборка в каталог, который читает сервер.
- **`apps/server`** (`@polka/server`) — Express, каталог **`routes`** под `express-openapi`, статика UI, Swagger UI, healthcheck, SPA fallback.

### A2. Один сервер хостит API и UI

- API под префиксами фич (`/wxh`, …), **`/docs`**, **`/health`**, при необходимости отдача сырой спеки.
- Статика SPA + **fallback на `index.html`** для клиентских маршрутов, не пересекающихся с API.

### A3. OpenAPI + express-openapi + Swagger UI

- Канонический файл спецификации (например `openapi.yaml` в `apps/server`).
- Регистрация приложения через `express-openapi` с **`paths`** → `routes`.
- Развести в спеке пути под префикс `/wxh` для секции WxH.

### A4. Dockerfile

- Привести к реальному entrypoint после миграции; multi-stage; финальный образ запускает один процесс Node.
- `EXPOSE` / `PORT` согласовать с `packages/config`.

### A5. Документация и DX

- Корневой `README.md`: workspaces, `build.sh`, ссылка на этот план.

**Критерий готовности фазы A:** `npm install`, `npm run build`, `npm start` поднимает сервер; UI открывается; `/health`, `/docs`, OpenAPI в согласованном виде; `TAG=test ./build.sh` собирает образ.

## Фаза B — Секция 1: WxH placehold

Цель: API под **`/wxh/`** и визуальный конфигуратор ссылок во Vue (базовый URL + `PUBLIC_ORIGIN` из конфига при необходимости).

### B1. API (WxH)

1. Генератор в `packages/generator`: плейсхолдер с лимитами из значений, переданных с сервера из `config.placeholderMaxWidth` / `placeholderMaxHeight`.
2. В `routes` (express-openapi): операции `GET /wxh/{width}/{height}` и при необходимости `GET /wxh/{width}/{height}/{text}`; query: `bg`, `fg`, `rounded`, `font`, `gradient`; ответ `image/svg+xml`.
3. Описать всё в OpenAPI; ошибки 400 при выходе за лимиты (если поддерживает валидатор).

### B2. UI — общая оболочка (все секции)

Макет и навигация зафиксированы для всего приложения конфигураторов:

| Элемент | Описание |
|--------|----------|
| **Лейаут** | Основной контент в **центрированной колонке**: ориентир **`max-width: 72rem`**, по бокам **горизонтальные отступы** `clamp(1rem, 4vw, 2rem)`; вертикально — обычный поток блоков. |
| **Header** | Визуально аккуратный блок: **типографика** (читаемый шрифт, достаточный контраст, без «кричащего» капса). Отображаемое имя — **Polka**; при желании лёгкий визуальный акцент (вес/размер), без обязательного подзаголовка. Рядом или строкой ниже — **версия**. |
| **Версия** | Одна строка: **`version` из корневого `package.json` монорепо**, подставляется при сборке `packages/ui` (например `vite` + чтение JSON с `../../package.json` или `define`), без отдельной «версии UI». |
| **Ссылка на API** | В header или компактной строке под ним — ссылка **«Документация API»** (или эквивалент по-русски) на **`/docs`** (Swagger UI). |
| **Язык интерфейса** | **Только русский:** подписи вкладок, полей, кнопок, подсказок, заглушек. |
| **Табы секций** | Под header — **вкладки**: **WxH**, **Аватары**. Соответствуют маршрутам Vue Router (**`/wxh`**, **`/avatars`**) и SPA fallback на сервере. |
| **Контент вкладки** | В каждой вкладке — **свой мини-конструктор ссылки**: поля по параметрам API (path + query), блок **итоговый URL**, **копировать в буфер**, **превью** (например `img` с собранным URL). |

**Принятые ответы по UI (зафиксировано):** (1) версия — из монорепо; (2) оформление названия — «красиво», с упором на типографику, display name **Polka**; (3) ширина/отступы — как в таблице выше; (4) вкладка «Аватары» до API — **непринципиально**, по умолчанию вкладка доступна, внутри **нейтральная русскоязычная заглушка** («Раздел в разработке» / аналог); (5) ссылка на **API** — да; (6) язык — **только русский**.

### B3. UI — вкладка WxH (мини-конструктор)

- Поля: `width`, `height`, опционально текст (path или query — как договоримся в B1), `bg`, `fg`, `rounded`, `gradient`, при необходимости размер шрифта (`font`).
- Превью, копирование, базовый origin для абсолютных ссылок (dev / `PUBLIC_ORIGIN`).
- Собранный путь всегда с префиксом **`/wxh/`**.

### B4. UI — вкладка «Аватары» (зарезервировано)

- После появления API под отдельным префиксом (например `/avatar/…`) — тот же паттерн: поля из OpenAPI, URL, превью, копирование.
- До готовности бэкенда — по умолчанию **заглушка на русском** (вкладка активна, контент — краткое сообщение о разработке); допустимы другие UX-варианты без изменения общего макета.

**Открыто по продукту (при реализации B):** текст только в path или дублировать `?text=` для длинных строк; PNG в этой секции — только если явно включим в scope.
