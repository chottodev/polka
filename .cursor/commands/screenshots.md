# Скриншоты UI (Polka) для README

**Результат команды `/screenshots`:** сгенерировать и перезаписать **три** PNG в `images/` — `wxh.png`, `avatars.png`, `stock.png` — по сценариям ниже (при необходимости поднять `npm run build` / `npm start` и открыть UI в браузере).

Ниже — **эталонные настройки**, чтобы картинки в `images/` и в `README.md` оставались согласованными между релизами. Снимать после `npm run build` и `npm start` (или `./start.sh`), URL: `http://127.0.0.1:4700/...` или `http://localhost:4700/...`.

## Общие параметры

| Параметр | Рекомендация |
|----------|----------------|
| Окно браузера | **~1280×900** (или 1280×800), при необходимости **full page** скрин, если форма + превью не влезают в viewport |
| Масштаб страницы | 100% |
| Тема / локаль | по умолчанию (светлая) |
| **seed** | там, где в форме есть поле **seed** (аватары, сток), перед съёмкой задать **случайное** значение (например несколько букв/цифр), **не** оставлять пустым и не фиксировать одно и то же навсегда — так превью не залипает на старом кеше и варианты аватаров не повторяются |
| Сохранение | ровно три PNG в `images/`: `wxh.png`, `avatars.png`, `stock.png` (других копий не делаем) |

## 1. Вкладка **WxH** — `/wxh`

Цель: показать конструктор плейсхолдера и превью SVG.

- **Ширина / высота:** оставить разумные дефолты (например **400** и **300**), либо зафиксировать конкретную пару и не менять без обновления README.
- **Текст:** можно без подписи в превью или с коротким текстом в query — главное, чтобы выглядело читабельно.
- **Скрин:** вся карточка секции (форма слева, URL и превью справа), как в текущем README.

## 2. Вкладка **Аватары** — `/avatars`

Цель: показать **векторный** режим и узнаваемого персонажа.

1. Переключить режим на **«Векторные»** (не «По инициалам»).
2. **Тип / kind:** выбрать **`man`** (в UI — «Мужской») — зафиксировать на скрин; при смене типа обновлять и этот документ, и README, чтобы не путать читателей.
3. Остальное: размер (например **256** или **128**), палитра **Soft** (или другая, но стабильная). В **seed** ввести **случайную** строку на каждый прогон (см. общую таблицу выше).
4. Убедиться, что **превью** отображает векторного аватара; при необходимости чуть прокрутить страницу, чтобы в кадр попали табы + форма + превью.
5. **Скрин:** сохранить как `images/avatars.png`.

## 3. Вкладка **Сток** — `/stock`

Цель: показать форму стока, **итоговый URL** и **превью** после загрузки.

1. **Ключевое слово (поиск):** зафиксировать, например **`космос`** (латиница/кириллица — как договоримся; для единообразия в README удобно **один и тот же** запрос между съёмками).
2. **Ширина / высота:** например **640** и **360** (ландшафт, согласуется с ориентацией в Freepik).
3. **fit / format / quality:** `cover`, `webp`, `82` (или текущие дефолты в UI).
4. В **seed (опционально)** указать **случайное** значение на каждый прогон (см. общую таблицу), чтобы не подтягивалось устаревшее закешированное превью.
5. **Обязательно:** в `.env` задан **`FREEPIK_API_KEY`**, иначе превью уйдёт в заглушку — для маркетингового скрина лучше с реальным ключом.
6. Нажать **«Применить параметры и обновить превью»** и **дождаться** загрузки картинки (несколько секунд).
7. **Скрин:** сохранить как `images/stock.png`.

## 4. Обновление README

После замены файлов в `images/`:

- Проверить, что `README.md` ссылается на актуальные имена (`wxh.png`, `avatars.png`, `stock.png`).
- Кратко в коммите указать, что обновлены скриншоты (и при смене сценария — поправить этот файл, если поменялись тип аватара / ключевое слово / размеры; **seed** намеренно не фиксируем в документе — каждый раз случайный).
